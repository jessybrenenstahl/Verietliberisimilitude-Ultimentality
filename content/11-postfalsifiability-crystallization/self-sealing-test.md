# The Self-Sealing Test

## Definition

The **self-sealing test** is a diagnostic run against a claim, a page, the marking apparatus, or the whole wiki to detect whether it has become *self-sealing* — a structure that cannot be wounded by any encounter because every attempted refutation is rerouted into confirmation, absorbed as material, or redescribed as support without altering a single term, count, mark, authority, or action. Self-sealing is the operational signature of totalization ([[totalization-boundary]]): closure that has stopped paying the cost of correction. The test is the framework's demand to *leave it answerable* compiled into a runnable procedure — the point at which the injunction stops being a posture and becomes a checklist with typed verdicts and a required update path.

It is not a measure of whether the tested claims are *true*. It measures whether they are *reachable* — whether a possible result could still change them. A structure can be largely accurate and yet sealed; a structure can be wrong in a hundred places and yet fully open. The test isolates the second axis and reports only on it.

Run it after any major change to the wiki and after every change to the marking apparatus itself — the apparatus being the one component whose sealing would silently exempt everything downstream.

Required checklist (each item scored independently):

- Can a central term be deleted?
- Can a count change?
- Can a mark be demoted?
- Can a governing authority be replaced?
- Can a corrective layer block action?
- Can an objection survive without being redescribed as support?
- Can failure of this test alter the tested system?

Verdicts — one per item, never collapsed into a single opaque score:

```text
pass     — a result exists that would change the system on this axis
partial  — the change is possible in wording but blocked at routing or effect
fail     — no result on this axis can alter the system
untested — the axis has not been probed; not to be read as pass
```

The seventh item is the hinge: a diagnostic whose own failure changes nothing is itself sealed, and the test would then have to return `fail` on itself.

## Type and formal status

**E:** Derived, **CV** — a diagnostic *carving*. The seven questions are one contestable partition of the ways a structure can seal against correction; a sharper, shorter, or differently cut checklist, or a demonstrated seam the list does not probe, is the standing attack. The verdict scale (`pass / partial / fail / untested`) is likewise Derived, CV: a better-typed set of outcomes could replace it.

**A:** The test aspires to *detect actual self-sealing*. A false `pass` on a genuinely closed system, or a `fail` on a system that in fact remains reachable, is an inaccuracy of the instrument, not merely a disallowed reading — the mapping can be wrong, and its wrongness is the alethic exposure.

This is a **treatise-side extension, held contestable**. It carries no controlling authority, fixes nothing by fiat, and takes no exempt or unmarked tier. Promoting the checklist to a self-certifying gate — a test whose `pass` licenses belief — would be exactly the [[textual-nephilim]] the test exists to catch.

## What it regulates

The test regulates the characteristic *excess* of [[postfalsifiability]]: a system engineered to survive attack can convert survival into invulnerability. Every mechanism that lets the framework outlast refutation — scar records, invariant recognition, the refusal to count a failed attack as confirmation — can be turned inside out into a machine that never loses. The self-sealing test is the counter-pressure on that drift. Specifically it regulates:

- the slide from [[crystallization]] to fossilization — resistance produced by selected elimination versus resistance produced by refusal, authority, or semantic reclassification;
- the slide from effective to ornamental correction ([[effective-and-ornamental-answerability]]): criticism preserved, displayed, even summarized, while the governed process stays causally immune;
- the logging of a *failed attack as confirmation* — the move the [[outcomes-of-attack]] classification prohibits, and the most common way a wiki seals;
- capture of the corrective layer ([[capture-of-corrective-layer]]) by the very process it audits.

Each question is a probe aimed at one excess. "Can a mark be demoted?" tests the ratchet; "Can an objection survive without being redescribed as support?" tests the confirmation trap; "Can a corrective layer block action?" tests capture.

## What regulates it

The test is a content-bearing claim and is therefore marked like any other. What holds it open:

- **[[self-application]].** The seventh item forces the test onto itself. Applied to the self-sealing test, the questions must return live verdicts:

```text
central term deletable?      pass    — "self-sealing" can be recut or dropped
count changeable?            pass    — seven items is a CV carving, revisable
mark demotable?              pass    — CV, never promoted to exempt
authority replaceable?       pass    — the test entrenches none; its apparatus is swappable
corrective layer blocks?     pass    — a fail must halt "accept the change"
objection survives?          pass    — a valid seam it misses stands as a fail
failure alters system?       pass    — a self-fail demotes or rewrites the test
```

- **[[causal-falsifiability]].** A verdict counts only if it can change state. `pass` on "can a mark be demoted?" is empty unless a demotion has an actual pathway; the test inherits the requirement that its own outputs trace to changed content, marks, or links.
- **[[same-level-attack-rule]].** Each question must be posed at the level where the claim lives; a token-level objection scored against a type-level claim is a mis-posed probe, not a `fail`.
- **[[strengthening-without-absolutization]].** The instrument must be able to return `fail` *and* to force a demotion, not only to certify. A test that can only tighten is a one-way ratchet — itself a form of sealing.
- **[[semantic-closure-and-recursive-marking]].** The test may not appeal to an extra-symbolic auditor to license its verdicts; the audit stays inside the medium it audits, which is why closure and marking co-regulate rather than one standing above the other.

## Valid attack surface

A valid attack targets the *instrument*, at its level:

1. Exhibit a checklist **item that can never return `fail`** for any possible system — a vacuous probe that always passes and therefore measures nothing.
2. Exhibit a **genuinely sealed system that the checklist scores as open** — a false `pass`, i.e., a seam the seven questions do not reach.
3. Show the **verdict scale collapses** in practice — that `partial`, `untested`, and `pass` are used interchangeably, so the typing does no work.
4. Show the seventh item is **decorative** — that a `fail` on the test cannot, in the running system, alter the system.

Invalid moves, rejected for a stated type reason: attacking the *content* of a page the test flagged (that contests the page, not the diagnostic); offering the framework's survival of some prior attack as proof the test is unnecessary (a failed attack is not confirmation — [[outcomes-of-attack]]); demanding the test certify truth (category error — it certifies reachability, not correctness).

## What happens if isolated

Run the checklist **without a causal update path** and it becomes checklist theater: pages accumulate `pass` labels that no operation consumes, an audit whose findings route nowhere. This is ornamental answerability wearing the costume of rigor — and, tellingly, the exact condition the test is built to name, now instantiated *by the test itself*.

Run it **without closure** — reaching for an external tribunal to certify each verdict — and it recreates the regress it was meant to foreclose: who audits the audit, and who audits *that*, with no operation permitted to stop the stack. Isolated from [[outcomes-of-attack]], its verdicts map to no defined change and mean nothing. Isolated from [[causal-falsifiability]], `pass` degrades from a traced capability into a badge.

## What larger property emerges from the coupling

Coupled to a live update path — so a `fail` can delete a term, demote a mark, replace an authority, or inhibit an action — and to recursive marking — so the test is subject to itself — what emerges is **auditable openness**: not the *claim* that the system is unsealed but the demonstrated capacity to fail one's own audit and change because of the failure. This is the operational face of [[self-verifying-not-self-certifying]] — the system can verify that it is still reachable without certifying that its current account is complete. It is what keeps a [[crystallization|crystal]] distinct from a fossil under load, and what lets the [[kill-table]] and [[attack-surface-matrix]] remain instruments rather than trophies.

## What would actually kill the claim

The page dies by its own last question. The self-sealing test is refuted if it is shown to be **itself self-sealing** — concretely, if either:

- **no configuration** of the wiki could score `fail` on any item (the checklist is vacuous, always-`pass`, and therefore measures nothing); or
- a `fail` **provably cannot alter** the tested system — the verdict is logged, displayed, even acknowledged, while no term is deleted, no mark demoted, no authority replaced, no action inhibited.

Either condition makes the instrument ornamental, and an ornamental self-sealing test is the very object it was built to detect. A weaker-but-real kill also stands: exhibit a system that is demonstrably sealed on some axis which the seven questions unanimously score `pass` — a missed seam that demotes the carving until the checklist is recut to reach it.

Residue: the test cannot prove it has enumerated *every* axis of sealing. Its completeness is a standing CV claim, open to a demonstrated eighth question. A `pass` across all items is not a certificate of openness — only the absence, so far, of a found seal.

## Prohibited misreadings

- **Not a single score or grade.** The test outputs typed per-axis verdicts; a collapsed "openness rating" destroys the routing/effect distinctions the instrument exists to draw.
- **`pass` is not a truth certificate.** Passing means *not sealed on that axis*, never *correct* or *complete*. Reading `pass` as endorsement is [[self-verifying-not-self-certifying|self-certification]].
- **`untested` is not `pass`.** Silence on an axis is an open question, not a clean bill.
- **A `fail` is not a refutation of framework content.** It flags a *governance* defect — a welded seam — repairable by restoring the update path, not by rewriting the claims the seam guarded.
- **Running the test is not itself answerability.** The audit produces no causal force on its own; the update path must exist independently ([[effective-and-ornamental-answerability]]).
- **The test's survival of an attack is not confirmation.** A failed attack on the diagnostic is logged as a failed attack, never as evidence the wiki is open ([[outcomes-of-attack]]).
- **The test is not exempt from itself.** It carries no unmarked tier; a checklist that could not fail itself would already have failed itself.
